Button dropdowns

DesignAPI

What are button dropdowns?

Button dropdowns are buttons that reveal a list of actionable options.

A button dropdown consists of two main parts:

  • A button with a label that, when clicked, displays a list of options.
  • The list of options which, when one is selected, executes an action.

When to use

Button dropdowns are ideal for collapsing 3 or more options into a single button to conserve screen space.

They are most commonly seen within tables due to space restrictions, but they are not limited to tables.

Alternate considerations

  • Use buttons if you have a smaller number of options and also want to present choices to users without hiding action options.
  • Use a dropdown select if an option is not an action. A button dropdown should be used to execute an action, not to make a selection from a list.

Anatomy

Button

The shape that encloses the label and chevron.
The dropdown buttons will always be ZUI Blue 500. Follow the button guidelines for sizing and spacing specifications.

Button label

Text that defines what type of options are in the list.

Dropdown chevron

The action icon within the button that, when clicked, reveals the options list.

Option list

The grouping of related actions the user can select.

Option

A unique action a user can select.

Organizing the list

Sort the list in a logical order. For instance, put the most selected option(s) at the top.

Using icons

Icons may be used beside options; they should assist by making selection easier for users.

See icon design specs

Labeling

  • The button label should be an umbrella action or category that encompasses all options in the list. After reading a label, the dropdown options should meet a user's expectations.
  • Use action verbs and clear verb phrases for your labels. For example, Save to indicates additional options are available, while Save does not.
  • Use a dropdown button without a label to minimize footprint.
  • Ideally, button dropdowns should be clearly labeled. If there are circumstances that prevent that from happening, use an icon that clearly and unambiguously represents its list. There may be situations where the list in the button dropdown is too varied for a clear and unambiguous icon. In these situations, consider other options before using a generic icon (like a chevron).

Interaction behavior

Buttons have their own unique behavior that indicates the user's interaction resulted in a change. Button dropdowns follow the same guidelines as regular buttons when it comes to behavior and interaction.
