Focused state

Focused state is used to minimize distractions for the user while they are completing forms or complex tasks. Focused state also keeps the action buttons in a standard state on the screen, even when scrolling. It is also the ideal pattern to use when walking the user through a complex step flow. The Focused state specs contains information on sizing and spacing.

Focused state types

Focused state - standard

  • Focused state uses a blue bar that is sticky to the bottom of the Top bar
  • The page title is located on the left side within the blue bar
  • Action buttons are located on the right within the blue bar
  • While the content on the page follow our max-content widths, the page title and buttons float to the far left and right using the entire width of the screen.

Focused state - step flow

  • Focused state step flow uses the same sticky blue bar at the top of the page
  • The progress indicator is located on the left within the blue bar and follows the format # of #: Step title
  • Directional buttons are located on the right within the blue bar
  • While the content on the page follow our max-content widths, the page title and buttons float to the far left and right using the entire width of the screen

Focused state - mobile

The mobile Focused state is a bit different from the desktop version.

  • The Top bar should disappear completely when entering a Focus state on a mobile device
  • The blue bar remains sticky to the top of the page
  • The cancel or back directional button should move to the left side of the screen
  • The title or progress indicator should be centered and will be truncated if it is too long
  • The save or forward directional button should be located on the right

Behavior

The main purpose of the focus state is to reduce distractions and increase the screen real estate for the actions. The Top bar and Side nav should remain available in a focused state to not limit the user in their ability to navigate out of the current workflow when necessary. To achieve that, the following behavior should be used:


  • Zywave Top bar behavior should remain unchanged
  • The Side nav should automatically collapse when the user enters the Focus state
  • Aside from being collapsed automatically, no other behavior of the Side nav should change
    • Hover behavior remains the same
    • Users should still be able to expand the Side nav if they wish - the content within the page will respond to this change, moving to the right or changing size to make room for the Side nav.
  • The Focused state header is sticky and should remain flush up against the bottom of the Top bar, even as the user scrolls down the page to keep the actions in a consistent place on the screen
  • Depending on the task at hand and how the data is being saved, a warning message may need to be displayed before a user navigates away to prevent loss of information

Best practices

  • Focused state can be used for complicated forms that are longer than 5 input fields
  • Focused state can be used for step flows where one or more of the steps is complex
  • There should only be one save or continue button when using a Focused state
    • The primary button in the Focused state blue bar should be your only save
    • This eliminates confusion around what portions of the page are saving when different buttons are being pressed
  • Focused state can contain expanders that are editable
  • Editable expanders should not include their own save action
  • Be consistent with your expanders. Do not combine editable and not editable expanding components within one UI
